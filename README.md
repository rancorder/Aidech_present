# Aidech Management｜台本操作型 Web Slides

営業商談用のWebスライドです。**発表者ビュー（手元・台本あり）** と **顧客表示（共有用・スライドのみ）** を分離しているため、画面共有しても手元の台本（カンペ）は相手に見えません。WSE presentation と同じ2画面分離アーキテクチャです。

## 使い方（画面共有での商談）

1. `index.html`（＝デプロイURL）を Chrome / Edge で開く（これが発表者ビュー）
2. 右上の「**顧客表示を開く**」を押す → 別ウィンドウが開く
3. Web会議では、その**顧客ウィンドウだけ**を画面共有する
4. 手元の発表者ビューで台本（一言目／補足説明／押さえるポイント／相手に投げる質問）を見ながら、次へ/前へ、またはカンペのクリックで進行
5. 両ウィンドウのスライドは自動で同期します

> ⚠ 発表者ビュー側を共有すると台本が見えます。共有するのは必ず顧客ウィンドウだけにしてください。

## 操作キー

- `→` / `Space`：次へ
- `←`：前へ
- `1`〜`9`：番号のスライドへ
- `R`：動画を最初から
- `F`：全画面

## スライドの編集

中身はすべて `slides.json` にあります。1スライド = 1オブジェクトで、以下の項目を持ちます。

| キー | 内容 |
|---|---|
| `id` | 通し番号 |
| `title` | スライドタイトル |
| `image` | 表示画像のパス（例 `images/1.png`） |
| `video` | 任意。動画のパス（例 `images/1.mp4`）。ファイルが存在すれば自動で動画再生、無ければ画像表示 |
| `first` | 一言目（最初に言うこと） |
| `talk` | 補足説明 |
| `point` | 押さえるポイント（手元メモ。読み上げ用ではない） |
| `ask` | 相手に投げる質問 |

台本は商談で読み上げる前提のたたき台です。実際の言い回しに合わせて自由に書き換えてください。

## 動画（MP4）対応

各スライドに `images/1.mp4` のように動画を置くと、そのスライドだけ自動で動画に切り替わります（H.264 / 音声なし / 1920×1080 / 5〜8秒 推奨）。無ければ画像のままです。`video` 未指定でも `images/<id>.mp4` を自動で探します。

> 動画を置いていない場合、ブラウザのコンソールに `<id>.mp4` の 404 が出ますが、これは「動画があるか探して、無いので画像にフォールバックした」という正常動作です。気になる場合は `video` フィールドを消してください。

## ファイル構成

```
index.html        最小シェル
app.js            2画面分離・同期・MP4自動検出・台本表示のロジック
style.css         Aidechテーマ（navy / orange / paper）+ WSEレイアウト
slides.json       スライド内容と台本
images/1.png 〜 10.png   スライド画像
vercel.json       SPAルーティング
```

## デプロイ（Vercel）

1. Vercel でこのリポジトリを Import
2. Framework Preset：`Other`、Build Command 空欄、Output Directory 空欄
3. Deploy

同期は同一オリジン（同じデプロイURL）の別ウィンドウ間で動作します。

## 注意

画像は必ず `images/` 配下に同梱してください（Google Drive直リンク参照は使いません）。
